# Trademark and brand-use policy

The **code** in this repository is MIT licensed — see [LICENSE](LICENSE). The **Bannerwright name, logo mark (the `[ ▬ ]` glyph), wordmark, and marketing-site copy** are not covered by that licence. They belong to Dawid Kawalec.

This is the [Plausible / PostHog / Sentry-style](https://plausible.io/docs/branding/) split: take the code, fork it, run it — but don't pretend your fork is the original.

## You can

- Use the Bannerwright name and logo to refer to the project itself ("Bannerwright is an open-source AI workshop for HTML banners").
- Link to https://bannerwright.com with the name and logo.
- Mention Bannerwright in talks, blog posts, screenshots of your setup, comparison tables, and reviews.
- Use the logo in articles and content that talk about the project, with the proportions and clear space intact.
- Fork the code and run a fork under a **different name and brand**.

## You can't (without written permission)

- Run a hosted service named "Bannerwright" or anything confusingly similar.
- Sell a product that uses the Bannerwright name, logo, or wordmark.
- Use the name or logo in a way that implies endorsement, sponsorship, or affiliation that doesn't exist.
- Modify the logo glyph and present it as the Bannerwright mark.

## Forks

You're explicitly welcome to fork. If you're running a fork in production, please pick a name that's clearly your own — both to respect the trademark and to avoid confusing users who hit your instance expecting Bannerwright's behaviour.

If your fork diverges meaningfully from upstream, change the visible brand strings (`Bannerwright` in the nav, footer, login screen, OG image, etc.) so people know they're looking at your project, not this one.

Reach out at **hello@kawalec.pl** before doing anything you're unsure about. The default answer is "let's talk", not "no".
